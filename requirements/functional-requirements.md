# 股票交易系统 Functional Requirements（对标 TradingView Pro / 富途）

## 1. 文档目标与范围

本文档用于定义一款跨平台股票交易系统的功能需求（Functional Requirements），产品能力参考 **TradingView Pro** 与 **富途客户端** 的核心体验，并支持通过后台对接主流券商接口（富途、Firstrade、IBKR 等）实现统一交易。

- 客户端优先级：**Web > iOS > macOS > Android**。
- 系统目标：提供从行情发现、分析决策、下单执行、持仓风控到复盘协作的完整交易闭环。

---

## 2. 竞品能力分析

## 2.1 TradingView（Pro 级）能力拆解

### 2.1.1 核心优势
1. **图表能力极强**：多图联动、多周期切换、丰富绘图工具、指标生态、脚本扩展（Pine Script）。
2. **社区与内容驱动**：策略分享、观点发布、指标脚本市场、社交互动。
3. **多市场数据覆盖**：股票、期货、外汇、加密货币等统一图表体验。
4. **Alert 自动化能力**：基于价格、指标、脚本条件的实时提醒与 webhook 联动。

### 2.1.2 典型功能模块
- 图表引擎：K 线、分时、Heikin-Ashi、Renko 等；多窗口布局。
- 指标/策略：内置技术指标、策略回测、脚本编辑器。
- Watchlist：自定义分组、排序、筛选、联动图表。
- Alert：条件触发、通知渠道（App 推送/邮件/Webhook）。
- 市场扫描器：按技术条件筛选标的。
- 社区内容：观点流、脚本分享、关注体系。

### 2.1.3 局限（对新系统启示）
- 不同券商实盘执行链路体验不统一。
- 对中文用户和本土券商生态的深度整合相对有限。
- 交易与社区、图表之间在某些场景切换成本较高。

## 2.2 富途客户端能力拆解

### 2.2.1 核心优势
1. **交易闭环成熟**：开户、入金、交易、资产、财务报表、客服流程完整。
2. **港美股本地化体验强**：盘口、逐笔、资讯、财报、社区（牛牛圈）等贴近交易用户。
3. **账户与风控体系完善**：融资融券、保证金、风险提示、交易权限管理。
4. **多终端一致性较高**：行情与交易功能在不同端协同较好。

### 2.2.2 典型功能模块
- 行情页：分时/K 线、盘口、资金流、异动榜单。
- 交易页：下单、撤改单、条件单、账户资产、成交回报。
- 资讯页：快讯、研报、财报、公告、日历。
- 社区页：讨论、观点、问答。
- 账户页：资金、持仓、盈亏、报表、设置。

### 2.2.3 局限（对新系统启示）
- 图表高级分析深度通常不及 TradingView 的开放脚本生态。
- 若用户持有多券商账户，统一交易体验与跨账户分析存在提升空间。

## 2.3 结论：目标产品差异化方向

目标系统应融合两者优势：
- **TradingView 级分析能力** + **富途级交易闭环能力**；
- 通过多券商聚合实现“一个前台，多账户执行”；
- 以 Web 为核心先发，移动与桌面端逐步一致化。

## 3. 产品总体功能蓝图

### 3.1 一级模块
1. 用户与账户体系
2. 市场数据与行情中心
3. 图表分析与策略研究
4. 选股/扫描与自选管理
5. 交易执行与订单管理
6. 资产持仓与绩效分析
7. 风控与预警中心
8. 资讯、事件与社区
9. 消息通知与任务中心
10. 券商接入与交易网关
11. 管理后台与运维审计
12. 跨平台客户端能力层

### 3.2 角色定义
- 访客用户：仅浏览公开行情（受行情授权限制）。
- 注册用户：可配置自选、提醒、图表模板。
- 交易用户：绑定券商账号后可下单交易。
- 专业用户（Pro）：使用高级图表、策略、回测等能力。
- 管理员/运营：内容审核、风控配置、系统监控。

## 4. 详细 Functional Requirements（按模块）

## FR-01 用户与账户体系

### FR-01-01 注册登录
- 支持邮箱、手机号、第三方登录（Apple/Google/微信可配置）。
- 支持多因子认证（MFA）与设备信任机制。
- 支持异地登录检测与风险验证（验证码/二次确认）。

### FR-01-02 用户资料与偏好
- 用户可管理昵称、头像、语言、时区、主题（深色/浅色）。
- 可配置默认市场、默认图表模板、默认下单参数。

### FR-01-03 订阅与权限
- 支持免费版/专业版权限控制（指标数量、Alert 数量、布局数量等）。
- 支持会员到期提醒、续费、降级策略。

---

## FR-02 市场数据与行情中心

### FR-02-01 多市场行情接入
- 支持美股、港股、A 股（若合规接入）、ETF、期权（阶段性）、期货（可选）。
- 支持实时、延时、收盘数据三级授权策略。

### FR-02-02 行情展示能力
- 行情列表支持代码、名称、最新价、涨跌幅、成交量、换手率等字段。
- 支持列表排序、固定列、自定义字段模板。
- 支持板块/概念/行业分组浏览。

### FR-02-03 Level1/Level2
- 支持基础买卖档位（Level1）显示。
- 对接支持 Level2 的券商或数据源时，展示深度盘口、逐笔成交。

### FR-02-04 数据质量要求
- 行情推送延迟与丢包率须可监控。
- K 线聚合需支持 1m/5m/15m/1h/1D/1W/1M 周期一致性校验。

---

## FR-03 图表分析与策略研究

### FR-03-01 图表基础
- 支持分时、蜡烛图、面积图、OHLC、Heikin-Ashi 等图表类型。
- 支持前复权/后复权/不复权切换（按市场规则适配）。
- 支持多图布局（1/2/4/6/8/9/12）与联动（标的、周期、十字光标）。

### FR-03-02 技术指标
- 提供 MA、EMA、MACD、RSI、BOLL、KDJ、VWAP 等内置指标。
- 支持指标参数编辑、叠加、保存为模板。
- 支持用户创建自定义指标（脚本化能力，P1 可先限制函数集合）。

### FR-03-03 绘图工具
- 支持趋势线、平行通道、斐波那契、江恩线、文本标注、形态标注。
- 支持对象吸附、锁定、隐藏、分组与跨设备同步。

### FR-03-04 回测与策略
- 支持基础策略回测：收益、回撤、胜率、盈亏比、交易次数。
- 支持参数优化（网格搜索基础版）与策略版本管理。
- 支持回测数据导出（CSV/JSON）。

### FR-03-05 Alert 与自动化
- Alert 触发条件支持：价格阈值、指标交叉、策略信号、成交量异常。
- 通知渠道支持：App Push、邮件、Webhook。
- 支持 Alert 冷却时间、防抖机制与触发历史查询。

---

## FR-04 选股/扫描与自选管理

### FR-04-01 自选列表
- 支持多个自选分组、拖拽排序、批量导入导出。
- 支持标签系统（如“财报季”“短线观察”“价值投资”）。

### FR-04-02 条件扫描器
- 支持基本面条件（市值、PE、EPS 增长）、技术面条件（均线突破、RSI 区间）。
- 支持盘前、盘中、盘后不同扫描模板。
- 扫描结果可一键加入自选并创建 Alert。

### FR-04-03 热榜与异动
- 提供涨跌幅榜、成交额榜、换手率榜、资金流榜。
- 异动事件（放量、急涨急跌、跳空）需可订阅提醒。

---

## FR-05 交易执行与订单管理

### FR-05-01 多券商统一交易入口
- 支持绑定多个券商账户（富途、Firstrade、IBKR），统一查看可交易标的。
- 根据标的与账户权限自动过滤可下单账户。

### FR-05-02 下单能力
- 订单类型：限价单、市价单、止损单、止盈止损单、条件单（阶段实现）。
- 支持盘前盘后（依据券商支持）与有效期（DAY/GTC）。
- 支持一键反手、快速平仓（高级权限控制）。

### FR-05-03 订单生命周期
- 展示待提交、已提交、部分成交、已成交、已撤销、拒单、失效状态。
- 支持撤单、改单、批量操作。
- 对拒单返回标准化错误码与可读原因。

### FR-05-04 成交回报与对账
- 实时回报成交明细，支持手续费、税费拆解展示。
- 日终对账：系统成交与券商成交一致性校验。

---

## FR-06 资产持仓与绩效分析

### FR-06-01 资产总览
- 展示总资产、现金、持仓市值、当日盈亏、累计盈亏、购买力。
- 多币种账户需支持汇率折算（可选基准币）。

### FR-06-02 持仓分析
- 展示持仓成本、现价、浮盈亏、收益率、仓位占比。
- 支持按账户/市场/行业/策略标签维度聚合。

### FR-06-03 绩效报表
- 支持日/周/月/年维度收益曲线。
- 指标包含夏普比率、最大回撤、波动率、胜率。
- 支持导出报表（PDF/CSV）。

---

## FR-07 风控与预警中心

### FR-07-01 交易前风控
- 订单金额、单票仓位、杠杆倍数、可用资金、交易时段校验。
- 命中规则时支持拦截或二次确认。

### FR-07-02 交易中风控
- 监控异常成交、重复下单、接口超时重试。
- 检测价格滑点超阈值并提示。

### FR-07-03 交易后风控
- 提供日内损失阈值、最大回撤阈值告警。
- 支持强平建议（仅提示）与风控日报。

---

## FR-08 资讯、事件与社区

### FR-08-01 资讯中心
- 聚合快讯、公告、财报、研报、宏观数据日历。
- 支持按标的订阅、关键词订阅、事件提醒。

### FR-08-02 财务与公司信息
- 展示利润表、资产负债表、现金流、估值指标。
- 提供同比/环比趋势视图。

### FR-08-03 社区能力（阶段性）
- 用户可发布观点、图表截图、策略想法。
- 支持评论、点赞、收藏、举报。
- 支持内容审核与风控关键词过滤。

---

## FR-09 消息通知与任务中心

### FR-09-01 通知渠道
- 站内信、App Push、邮件、Webhook。
- 支持通知优先级与免打扰时段。

### FR-09-02 通知类型
- 价格提醒、订单状态提醒、风控提醒、系统公告。
- 消息支持已读/未读、搜索、归档。

---

## FR-10 券商接入与交易网关

### FR-10-01 券商适配器架构
- 每个券商通过 Adapter 接入，屏蔽 API 差异。
- 统一抽象：账户、持仓、订单、成交、资金、行情权限。

### FR-10-02 接口可靠性
- 支持重试、幂等键、防重复提交。
- 关键链路（下单/撤单）需有追踪 ID 与审计日志。

### FR-10-03 安全与合规
- API 凭据加密存储与轮换。
- 对敏感操作进行操作审计与异常告警。
- 符合目标市场的监管与数据授权要求（按部署地区配置）。

---

## FR-11 管理后台与运维审计

### FR-11-01 运营配置
- 管理市场开关、功能灰度、会员套餐、Banner 与公告。

### FR-11-02 风控运营
- 配置风控规则、黑白名单、限额策略。
- 审核异常订单与人工干预记录。

### FR-11-03 可观测性
- 指标监控：行情延迟、订单成功率、接口耗时、错误率。
- 日志检索与告警分级（P0/P1/P2）。

---

## FR-12 跨平台客户端能力层

### FR-12-01 平台优先级与功能策略
- **Web（P0）**：首发全量核心功能（行情、图表、下单、资产、Alert）。
- **iOS（P1）**：覆盖高频交易场景与通知闭环。
- **macOS（P1）**：强化多屏交易与专业分析布局。
- **Android（P2）**：补齐核心闭环并保证与 iOS 功能等价。

### FR-12-02 一致性要求
- 统一设计系统与组件库，保证交互一致。
- 图表模板、自选、提醒、布局跨端同步。

### FR-12-03 性能与体验
- 行情首屏加载、图表缩放流畅度、下单链路反馈时间需达到可用阈值。
- 弱网环境下支持断线重连与降级展示。

---

## 5. 优先级与里程碑建议（MVP → Pro）

## 5.1 MVP（建议 0~6 个月）
- Web 端：行情、自选、基础图表、基础下单、订单/持仓/资产、价格提醒。
- 券商：优先接入 1~2 家（建议 IBKR + 富途）。
- 风控：交易前基础规则。

## 5.2 Pro 阶段（6~12 个月）
- 高级图表、策略回测、自定义指标脚本。
- 扫描器、事件驱动提醒、更多订单类型。
- iOS/macOS 深化与多券商扩展。

## 5.3 扩展阶段（12 个月+）
- 社区生态、策略市场、组合分析、机构化风控。
- Android 全量增强与跨市场衍生品支持。

---

## 6. 非功能性补充（与功能需求强相关）

- 可用性：交易时段核心链路高可用。
- 安全性：账户安全、接口凭据安全、审计可追溯。
- 合规性：行情授权、交易监管、数据隐私合规。
- 可扩展性：券商适配器与指标脚本能力可扩展。

---

## 7. 验收建议（功能视角）

1. 用户可完成“登录→看行情→画图分析→下单→查看持仓盈亏→设置提醒”完整闭环。
2. 同一用户绑定两个以上券商账户后可在统一界面完成交易并追踪订单状态。
3. Alert、订单状态、风控告警可通过至少两种通道送达。
4. Web 与移动端核心功能行为一致，关键数据同步正确。
